# Product Tree Extraction

A systematic protocol for discovering every sellable product embedded in a single document, framework, or body of expertise.

## When to Activate

- User asks what products they can create from their knowledge, document, or framework
- User wants to monetise expertise, IP, or a body of work
- User has a whitepaper, methodology, course, or consulting framework and wants to know what it contains commercially
- User asks how to productise, package, or sell their intellectual property
- Any scenario where the question is what is already in here that a buyer would pay for

## What This Skill Does

Product Tree Extraction takes any source document with structural density and identifies every embedded product it contains. Not through brainstorming or ideation, but through systematic structural extraction.

The protocol identifies products across four tiers:

- **Tier 1 - Diagnostic Instruments:** Tools that produce a score, classification, or gap analysis. The buyer applies them and gets a result.
- **Tier 2 - Assessment Frameworks:** Structured methodologies for evaluating a system, decision, or position. Broader than diagnostics.
- **Tier 3 - Quantitative Models:** Anything with numerical logic - formulas, calculators, pricing models, scoring algorithms.
- **Tier 4 - Strategic Architectures:** Complete systems - governance structures, operational frameworks, venture designs, institutional blueprints.

## Preview Method

When activated, perform a **Pass 1 Component Scan** on the user's source material:

1. Read the source document completely
2. Identify every discrete structural element: frameworks, models, diagnostics, taxonomies, classification systems, decision trees, scoring rubrics, process sequences, templates, checklists, assessment instruments, or named methodologies
3. For each element, write one sentence describing what it enables the user to DO (not what it is about)
4. Apply the standalone test: could a buyer use this element independently, without the rest of the source document, to produce a result they would pay for?
5. Present the inventory with tier classifications

This preview pass typically yields 5-15 elements from a single document.

## Limitations of This Preview

This skill provides Pass 1 (Component Scan) and basic Tier Classification. The complete protocol includes five passes that typically yield 20-50+ products from a single source document:

- Pass 1: Component Scan (included above)
- Pass 2: Full Tier Classification with pricing logic
- Pass 3: Reassembly, Bundling, and Derivative Extraction
- Pass 4: Dependency Mapping across all products
- Pass 5: Build Sequence, phased production plan, and final pricing

Passes 3-5 typically increase the product inventory by 30-60% above the Pass 1 yield through bundle architecture, derivative products (ND editions, implementation kits, audits), and cross-tree connections.
